🤔 What is Hot Module Reloading?

HMR allows you to update your application code without refreshing the page, while preserving your application state.

Benefits:

✅ No more losing form inputs
✅ Keep your scroll position
✅ Maintain component state
✅ Instant feedback on changes
✅ Faster development cycle 

// Example: Edit this file while the counter is at 42
// When you save, the counter will STILL be at 42!

function incrementCounter() {
    const counter = document.getElementById('counter');
    const current = parseInt(counter.textContent);
    counter.textContent = current + 1;
}

⚙️ Technical Details

How It Works:

1. File Watcher - Node.js watches your files for changes
2. WebSocket - Server notifies browser of changes
3. State Preservation - App state saved to sessionStorage
4. Hot Swap - CSS updated without reload
5. Smart Reload - JS/WASM triggers full reload with state restore 

// HMR Client connects to server
const hmr = new HMRClient();
hmr.connect(3002);

// Automatically preserves state before reload
hmr.preserveState();

// Restores state after reload
hmr.restoreState();
